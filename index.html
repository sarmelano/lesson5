1) git init = создать локальный репозиторий/подключить папку к git. 2) git status = проверить состояние 3) git add .расширение .расширение = добавить в git / зафиксировать внесённые изминения ( добавить все хтмл = *.html) git checkout .расширение = вернуть в исходное состояние git commit -m "комментарий" = синхронизировать/добавить файл в репозиторий git git commit -am "комментарий" = синхронизировать/добавить файл в репозиторий git одновременно с добавлением (git add + git commit) git diff = что именно было изменино/внесено в файлы (сравнение с прошлой версией - почему они стали modified) git log = показать всю историю изминений в репозитории "покоммитно" с номерами коммитов git log -n кол-во = показать количество последних изминений в репозитории "покоммитно" с номерами коммитов git revert ...номер коммита... = откатить данное изминение/коммит (после внесения поправок требует заново добавление и коммита) также можно сделать повторно git revert ...номер коммита... -чтобы восстановить откат на место (+1/0) вместо (+1/-1) git checkout -b "имя ветки" = создать ветку для редактора и переключится сразу на неё git checkout "имя ветки на котор хотим переключится" = переключение между ветками git merge ....имя бранча с которого хотим загрузить изминения к себе git branch -D имя ветки которую хотим удалить(влитую) git remote add origin https://github.com/sarmelano/lesson5.git = добавить в удаленный репозиторий git push -u origin master(или свой бранч) = подтвердить добавление в уд. репозиторий все изминения ----> git add ------> git commit -m "..."
